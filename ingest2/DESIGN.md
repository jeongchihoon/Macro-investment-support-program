# FinVision 파이프라인 설계서 (ingest2 북극성 문서)

> 이 문서는 사용자가 제공한 설계 narrative를 구조화해 보존한 것이다. **변경은 합의 후 이 문서에 반영**하고 `PLAN.md`의 결정 로그에 기록한다. 실제 빌드 순서·결정 상태는 `PLAN.md` 참조.

## 0. 한 줄 정의

FinVision은 뉴스를 많이 모아 보여주는 서비스가 **아니다**. 흩어진 정보(뉴스·공시·시장데이터)를 모아 **"오늘 투자자가 실제로 봐야 할 시장 재료 Top 10"**으로 정제하는 *편집국*이다. 출력 단위는 기사가 아니라 **투자 판단 단위(Top Market Item / 카드)**.

## 확정된 스코프 (2026-06-22)

| 항목 | 결정 |
|---|---|
| 시장 | **미국 단독** (한국발 예외만 language 필드로 대비) |
| 소스 | **SEC 공시 · 뉴스 API · RSS · 시장데이터** |
| 제외 | **텔레그램**(향후), **언론사 크롤링**(저작권·파서 취약성, RSS가 헤드라인 커버) |
| LLM | Gemini (다운스트림 AI 분석 단계) |

## 전체 흐름

```
[외부 소스] 뉴스 API / RSS / SEC 공시 / 시장데이터
      ↓
1. 수집        소스별 어댑터로 가져옴
2. 원본 저장   나중에 검증 가능하게 그대로 보관 (보험)
3. 정규화 저장 모든 소스를 공통 형식(NewsItem)으로 변환
4. 1차 필터    명백히 쓸모없는 것 제거 (+탈락 사유 기록)
5. 분류        기업·종목(직접/간접)·이벤트 유형 부착
6. 중복 제거   같은 사건을 하나로 묶음 (의미 기반)
7. 후보 생성   시그널 후보 / 스토리 후보 (한 풀에서 경쟁)
8. AI 분석     볼 가치·관련 종목·중요성 분석
9. 최종 편집   신뢰도·중복·리스크·법적표현 점검, 랭킹
      ↓
[Top Market Items] 오늘 볼 핵심 카드 5~10개 → 사용자 화면 → 피드백
```

**ingest2가 담당하는 범위 = 1~4 (+5 경량 분류는 경계, PLAN에서 결정).** 6단계(중복제거) 이후는 다운스트림(기존 `src/` 또는 후속 작업).

## 1. 소스와 신뢰도 등급

소스는 한 종류가 아니다. 부류별로 신뢰도가 다르고, 이 등급이 다운스트림 랭킹까지 흐른다.

| 등급 | 소스 | 역할 | 본 빌드 |
|---|---|---|---|
| 1 | **SEC EDGAR / IR / 실적자료** | 팩트 기준점. "누가 말했다더라"가 아니라 공식 제출 자료 (8-K, S-1, M&A, CEO/CFO 교체, 실적) | ✅ |
| 2 | **뉴스 API** (Finnhub/Polygon/Benzinga) | 깔끔한 구조(제목/요약/시간/언론사/URL/티커/카테고리). 크롤링보다 안정적. 단 호출제한·실시간성·중복 주의 | ✅ |
| 3 | **RSS** (Yahoo/CNBC/MarketWatch/Google News) | "새 글 알림 목록". 본문 부족 → 발생 *신호*로 사용, 필요시 원문 확인 | ✅ |
| 4 | 언론사 크롤링 | 사이트 구조 변경 시 파서 깨짐, 유료벽·약관·봇차단·저작권 부담 | ❌ 제외 |
| 5 | 텔레그램 | 뉴스 아닌 *레이더*(소문 확산·종목 급언급 감지). 단독 근거 불가 → 공시/뉴스+주가반응+다채널 확산으로 검증 후 승격 | ❌ 향후 |

**시장 데이터**(주가·거래량·프리/애프터마켓·상대거래량·섹터ETF·금리·달러·유가·VIX)는 뉴스가 아니다. 뉴스가 실제로 시장을 움직였는지 확인하는 **보강(enrichment)** 용도 → 별도 `MarketSnapshot`.

## 2. 수집: 어댑터 패턴

모든 소스를 같은 방식으로 가져오지 않는다. **소스별 전용 수집기(어댑터)**가 각자 가져오되, 안으로 들어오면 **공통 형식**으로 정리한다. (물류센터 비유: 입고 방식은 달라도 창고 안에선 같은 상품코드로 관리.)

- 인터페이스: `BaseCollector.fetch()` = 원본 수집, `.normalize()` = 공통 `NewsItem` 변환.
- 새 소스 추가 = 어댑터 하나 + registry 등록. 파이프라인 본체 수정 없음.

## 3. 저장: 2단계

1. **원본 저장** — 수집 직후 소스가 준 그대로 보관(JSON/HTML/텍스트). 필터 오작동·AI 오요약·출처확인·재분석·"이 카드 근거 뭐야?" 대응을 위한 **보험**.
2. **정규화 저장** — 공통 형식으로 변환: 제목·요약·본문·URL·원문URL·출처·발행시간·수집시간·언어·카테고리·관련기업·관련티커·이벤트유형·필터상태·탈락사유.

## 4. 1차 필터 (명백히 아닌 것만 빠르게 제거)

> 좋은 뉴스를 완벽히 고르는 단계가 아니다. "누가 봐도 아닌 것"을 치우는 단계.

- 너무 짧음/빈 본문/깨진 링크/발행시간 없음
- 너무 오래됨(24h+ 등) — 단 오늘 다시 중요해진 공시는 예외
- 무관 카테고리(스포츠/연예/여행/가십) 제거, 관련(Business/Markets/Economy/Tech/Finance/Energy/Healthcare/Policy) 유지
- 광고·홍보·조작성("지금 매수" "100% 수익" "리딩방")
- 미국 시장 무관 — **단 약하게**. 미국 기업명이 없어도 중요할 수 있음(예: SK하이닉스 HBM → NVDA/AMD/MU/TSM)

**버린 것은 사라지지 않는다** — `rejected_reasons`(too_short/old_article/no_market_relevance/sports_category/duplicate_url/spam_like) 기록. 초기엔 매일 탈락 샘플을 사람이 검수해 규칙 보정.

## 5. 2차 분류 (이게 무슨 얘기인지 부착)

- **기업명** 탐지 (문맥 필요: Apple=기업 vs 과일)
- **직접 티커** (Microsoft→MSFT)
- **간접 관련 종목** (MSFT AI 데이터센터 → NVDA/AMD/AVGO/MU/VRT/ETN) — 사전만으론 불가, 산업관계·AI 판단 필요 → 조심스럽게 표기(직접/간접 분리)
- **이벤트 유형** (실적/가이던스↑↓/M&A/IPO/CEO·CFO교체/소송/규제/공급망/AI CAPEX/주가급등락/공시/전략적투자) — 뉴스끼리가 아니라 *사건 중요도*를 비교하기 위함

## 6. 3차 중복 제거

같은 사건이 여러 경로(Reuters 원문/Yahoo 재배포/Investing 요약/CNBC 해설/번역기사)로 들어온다.

- **쉬운 제거**: 같은 URL/제목/원문링크/기사ID
- **어려운 제거**: 제목 달라도 같은 사건 (같은 기업·사건·시간대·가격움직임·동일 판단 제공?)
- **최종 기준 = 투자 판단 중복**: 사용자가 *같은 판단을 두 번* 하게 만들면 중복 → 하나로 합침

## 7. 4차 후보 생성

- **시그널 후보**: 구체적 단일 사건 (MSFT 하락 / NVDA 공급망 / Tesla 가이던스 하향)
- **스토리 후보**: 여러 시그널이 연결된 흐름 (빅테크 AI투자↑ → 클라우드 마진부담 / 반도체·전력 수요기대). 너무 추상적이면 안 됨.

## 8. 시그널 vs 스토리 — 별도 Top10 안 뽑는다

시그널·스토리·IPO·M&A·거시·공급망 후보를 **한 바구니**에 넣고 경쟁시켜 **Top 10 하나**만 낸다. 한 클러스터 안에서:

- **개별 시그널이 강하면** → 시그널을 메인, 스토리는 카드 내 배경으로 흡수 (예: "MSFT 하락 — AI 투자비 부담" + 배경: 빅테크 AI CAPEX 재평가)
- **여러 종목으로 퍼진 흐름이 강하면** → 스토리를 메인, 시그널을 하위 근거로 (예: "AI CAPEX 재평가" + 하위: MSFT↓/AMZN투자↑/MU수요/전력주↑)

## 9. AI 분석 (늦게, 비용 절감)

필터·중복제거 통과분에만 투입. 3단계:
1. **볼 가치 판단** — 미국 투자자에게 볼 가치가 있나? (평범/단순주가요약/구체사건/궁금증/종목연결)
2. **정리** — 직접·간접 종목, 이벤트, 섹터 연결
3. **카드 작성** — 제목·핵심요약·왜 중요·직접/간접 종목·리스크·출처. **투자조언 단정 금지** ("MSFT 매도" ✗ → "비용 부담 우려로 해석될 수 있음" ✓)

## 10. 최종 편집 (편집회의)

후보 점수화: 구체성·종목연결·출처신뢰도·신선도·시장반응·근거숫자·다중확인·실질 가치·비추상성·낮은 루머위험·비반복. **직접성 점수**(구체적 단일사건 > 추상적 전망), **신뢰도 점수**(SEC>Reuters>회사발표>뉴스API>재배포>루머). 마지막 **중복 재점검** + **법적 표현 점검**(매수/매도/확실/지금사야함 ✗ → "~로 해석될 수 있음/우려/추가확인 필요/투자권유 아님" ✓).

## 11. Top 10 기준

들어가려면: ①구체적 사건 ②종목·섹터 연결 ③신뢰가능 출처 ④오늘 볼 이유 ⑤생각할 거리 ⑥비중복 ⑦비추상 ⑧리스크 설명 가능. **억지로 10개 안 채운다** — 좋은 게 6개면 6개("오늘 핵심 재료 6개").

## 12. 시간축 갱신

- **실시간 긴급**: 대형 M&A 8-K, 대형주 3%+ 급락, FOMC, CPI 크게 상회, 대형 CEO 교체
- **1시간 갱신**: 수집→후보정리→중복제거→AI분석→Top Items 갱신
- **하루 정리**: 오늘 핵심 재료 / 반복 테마 / 내일 확인 포인트

## 13. 사용자 피드백

유용/별로/이미앎/추상적/종목연결약함/근거부족/관심없는섹터 → 다음 랭킹에 반영(개별 시그널 vs 거시 스토리 비중 조정 등).

## 14. 한 줄 정리

세상의 뉴스·공시를 다 모으는 게 아니라, **원본보관 → 정규화 → 필터 → 중복제거 → 종목/이벤트 분류 → AI분석 → 최종편집**을 거쳐 투자자가 *오늘 실제로 봐야 할* 핵심 시장 카드만 골라준다.
